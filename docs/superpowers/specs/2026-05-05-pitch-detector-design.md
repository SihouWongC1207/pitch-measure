# 音高测量 Android 应用 - 设计文档

## 概述

一款通用音高检测 Android 应用，通过手机麦克风实时采集声音，使用 YIN 算法分析音高，以明亮简洁的 Material 3 风格展示当前音名、频率、音准偏差和音高历史曲线。

## 技术栈

| 项目 | 选择 |
|------|------|
| 语言 | Kotlin |
| 最低 SDK | API 26 (Android 8.0) |
| UI 框架 | Jetpack Compose + Material 3 |
| 音频处理 | TarsosDSP (YIN 算法) |
| 构建工具 | Gradle (Kotlin DSL) |
| IDE | Android Studio |

## UI 设计

采用方案 B：大字体 + 弧形偏差指示器 + 全屏曲线图布局。

### 主界面元素

1. **顶部 - 音名显示**：超大字体居中显示当前音名（如 A4），下方显示精确频率（Hz）和音分偏差
2. **中部 - 弧形偏差指示器**：半圆弧形表盘，指示当前音高与标准音的偏差方向和幅度，中心点为"准确"
3. **底部 - 全屏音高曲线图**：显示最近 10 秒的音高变化曲线，带网格线辅助阅读，最新数据点高亮
4. **操作按钮**：「开始/停止」按钮 + 「设置」按钮

### 视觉风格

- 明亮简洁的 Material Design 3 风格
- 主色调：蓝色系 (#1565C0 / #42A5F5)
- 背景：白色 / 浅灰 (#FAFAFA)
- 准确指示：绿色 (#4CAF50)

### 模块职责

- **PitchDetector**：通过 AudioRecord 采集 PCM 音频数据，使用 TarsosDSP YIN 算法提取音高
- **PitchViewModel**：管理 UI 状态，将音高数据转换为音名/偏差信息，维护曲线历史数据
- **PitchScreen**：Compose 主界面，组合各 UI 组件
- **PitchCurveView**：自定义 Canvas 组件，绘制音高曲线

## 数据流

```
麦克风 → AudioRecord (44100Hz, 2048 buffer) → TarsosDSP YIN 分析
                                                    ↓
                                              频率(Hz) + 概率值
                                                    ↓
                                          换算：音名 + 音分偏差
                                                    ↓
                                            ViewModel 更新状态
                                                    ↓
                                        ┌───────────┴───────────┐
                                        ↓                       ↓
                              UI 更新音名/偏差显示    曲线图添加新数据点
```

### 音名换算

- 频率 → MIDI 编号：`69 + 12 × log₂(freq / 440)`
- MIDI 编号 → 音名：查表（如 69 = A4, 60 = C4）
- 音分偏差：与最近标准音的偏差，范围 ±50 音分

### 音频参数

- 采样率：44100 Hz
- 缓冲区：2048 采样（~46ms 一次分析）
- 算法：YIN（准确处理泛音，优于简单自相关）

## 权限与错误处理

### 权限

- `RECORD_AUDIO`：录制音频，运行时动态申请

### 错误处理

- 用户拒绝麦克风权限 → 显示友好提示，引导去设置开启
- 环境安静检测不到音高 → 显示"未检测到声音"而非空白
- 应用切到后台 → 自动停止音频采集，节省电量

### 生命周期

- 进入前台 → 可开始采集
- 退到后台 → 停止采集
- 用户点击开始/停止 → 手动控制
